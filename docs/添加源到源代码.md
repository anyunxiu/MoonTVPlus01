## 配置文件位置

源代码配置通过以下方式加载（优先级从高到低）：

1. **环境变量 `CONFIG_SUBSCRIPTION_URL`** - Base58 编码的配置 URL
2. **环境变量 `INIT_CONFIG`** - JSON 格式的配置（直接嵌入）
3. **管理后台** - 通过 Web UI 设置（数据库存储）

## 配置结构说明

### ConfigFileStruct 接口定义

```typescript
interface ConfigFileStruct {
  cache_time?: number;           // 接口缓存时间（秒），默认 7200
  api_site?: {
    [key: string]: {
      name: string;              // 源显示名称
      api: string;               // 源 API 地址
      detail?: string;           // 详情页 URL（可选）
    }
  };
  custom_category?: {
    name?: string;               // 分类显示名称（可选）
    type: 'movie' | 'tv';       // 类型：电影或电视
    query: string;               // 搜索关键词
  }[];
  lives?: {
    [key: string]: {
      name: string;              // 直播源显示名称
      url: string;               // M3U8 或其他直播 URL
      ua?: string;               // User-Agent（可选）
      epg?: string;              // 节目单 URL（可选）
    }
  };
}
```

## 添加播放源

### 方式一：使用 INIT_CONFIG 环境变量

最简单的方式是在部署时设置 `INIT_CONFIG` 环境变量。

#### Docker Compose 示例

```yaml
services:
  moontv-core:
    image: ghcr.io/mtvpls/moontvplus:latest
    environment:
      - USERNAME=admin
      - PASSWORD=your_password
      - NEXT_PUBLIC_STORAGE_TYPE=kvrocks
      - INIT_CONFIG='{
          "cache_time": 7200,
          "api_site": {
            "dyttzy": {
              "name": "豆瓣影视",
              "api": "http://api.dy01.cc/api.php/provide/vod"
            },
            "dyjx": {
              "name": "第一角色",
              "api": "http://www.1jx.net/api.php/provide/vod"
            },
            "kubo": {
              "name": "库博资源",
              "api": "http://api.kubo.top/api.php/provide/vod"
            }
          },
          "custom_category": [
            {
              "name": "华语电影",
              "type": "movie",
              "query": "华语"
            },
            {
              "name": "美剧",
              "type": "tv",
              "query": "美剧"
            }
          ],
          "lives": {
            "cctv": {
              "name": "央视频道",
              "url": "http://iptv.example.com/live/cctv1.m3u8"
            }
          }
        }'
```

#### 使用 .env 文件（需要处理转义）

在 `.env` 文件中添加：

```env
INIT_CONFIG={"cache_time":7200,"api_site":{"dyttzy":{"name":"豆瓣影视","api":"http://api.dy01.cc/api.php/provide/vod"}}}
```

## 环境变量方式

### Docker 环境变量设置

```bash
INIT_CONFIG='{...}'                    # JSON 配置
CONFIG_SUBSCRIPTION_URL=<base58_url>  # Base58 编码的配置 URL
```

### Cloudflare Workers 环境变量

在 `wrangler.toml` 中设置：

```toml
[env.production]
vars = { INIT_CONFIG = "{...}" }
```

### Vercel 环境变量

在 Vercel 部署设置中添加：

- 变量名：`INIT_CONFIG`
- 变量值：JSON 字符串（注意转义）

## 部署方法

### Docker 部署示例

```bash
docker run -d \
  -p 3000:3000 \
  -e USERNAME=admin \
  -e PASSWORD=your_password \
  -e NEXT_PUBLIC_STORAGE_TYPE=kvrocks \
  -e KVROCKS_URL=redis://moontv-kvrocks:6666 \
  -e INIT_CONFIG='{"cache_time":7200,"api_site":{"dyttzy":{"name":"豆瓣","api":"http://api.dy01.cc/api.php/provide/vod"}}}' \
  ghcr.io/mtvpls/moontvplus:latest
```

### 源代码部署

1. 克隆仓库：
```bash
git clone https://github.com/anyunxiu/MoonTVPlus01.git
cd MoonTVPlus01
```

2. 编辑 `src/lib/config.ts` 添加默认配置

3. 安装依赖并构建：
```bash
pnpm install
pnpm build
```

4. 启动应用：
```bash
INIT_CONFIG='{"cache_time":7200,"api_site":{"dyttzy":{"name":"豆瓣","api":"http://api.dy01.cc/api.php/provide/vod"}}}' \
pnpm start
```

## 注意事项

1. **JSON 格式**：确保 JSON 格式正确，可使用在线 JSON 验证工具
2. **转义字符**：在环境变量中使用 JSON 时需要正确转义双引号
3. **API 可用性**：添加的 API 源应该是可用的，定期检查源的有效性
4. **缓存时间**：`cache_time` 单位为秒，设置过小会增加服务器压力
5. **源唯一性**：`api_site` 中的 key 必须唯一
6. **分类 key**：`custom_category` 中 `type + query` 的组合必须唯一

## 常见问题

### Q: 如何使用配置订阅 URL？
A: Base58 编码你的 JSON 配置，部署到服务器，然后通过 `CONFIG_SUBSCRIPTION_URL` 环境变量指向该 URL。

### Q: 修改源代码后需要重新编译吗？
A: 不需要。只要修改了环境变量，重启容器即可生效。

### Q: 能同时使用 INIT_CONFIG 和管理后台配置吗？
A: 可以。INIT_CONFIG 是初始配置，管理后台修改会覆盖并保存到数据库。

### Q: 如何验证配置是否正确加载？
A: 访问 `/api/admin/config` 端点（需要登录），查看返回的 `SourceConfig` 字段。
